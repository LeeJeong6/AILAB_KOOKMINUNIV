# Lab Seminar Archive (GitHub Pages)

## 계획(Plan)

이 저장소는 연구실 스터디 아카이브를 GitHub Pages로 운영하기 위한 정적 웹 페이지입니다.
이번 개편의 목표는 학문적 분위기를 유지하면서도, 카테고리별 업로드 동선을 명확하게 만드는 것입니다.

1. 타이틀/메인 디자인 개편
- 논문 아카이브 느낌의 타이포그래피와 카드형 레이아웃 유지
- 메인 화면에 연구실 홈페이지(`https://ailab.kookmin.ac.kr/`) 대형 링크 배치
- 타이틀 비주얼 영역에 사용자 이미지 첨부 기능 추가

2. 카테고리 기반 업로드 UX 구현
- 카테고리를 3개로 고정
  - 논문 세미나 (`paper`)
  - 바이브코딩 (`vibe`)
  - 수학스터디 (`math`)
- 사용자는 먼저 카테고리에 입장한 뒤, 해당 카테고리 내부 폼에서 업로드
- 업로드 시 필수 메타데이터
  - 날짜(년-월-일)
  - 발표자
  - 제목
  - 파일(PDF/PPT/PPTX)

3. 저장 및 렌더링 전략
- GitHub Pages는 서버 업로드 API가 없으므로, 브라우저 `IndexedDB`에 저장
- 페이지 재방문 시 같은 브라우저에서는 업로드 기록 유지
- 카테고리별 목록 렌더링, 파일 열기/다운로드/삭제 지원

## 파일 구조

- `index.html`: 메인 페이지 및 업로드 로직(IndexedDB 포함)
- `2026-03-20-groupvit.html`: 기존 세미나 샘플 페이지
- `README.md`: 계획 문서
- `RESULT.md`: 결과 및 분석 문서(한국어)

## 사용 방법

1. `index.html`에서 카테고리 카드 중 하나를 눌러 입장
2. 해당 카테고리의 업로드 폼 입력
3. 저장 후 같은 카테고리 목록에서 파일 확인
4. 우측 상단 타이틀 이미지 영역에서 사진 선택/초기화 가능

## 주의 사항

- 현재 구현은 정적 사이트 제약으로 인해 서버 공유 업로드가 아닌 로컬 브라우저 저장 방식입니다.
- 모든 사용자에게 공통 반영되는 업로드 기능이 필요하면 백엔드/스토리지(예: GitHub API, Firebase, S3)가 추가로 필요합니다.
