## 🎯 빠른 배포 (5분 완료)

### 1️⃣ GitHub 저장소 생성 및 업로드

```bash
cd /raid/lab-seminar

# Git 초기화
git init

# 파일 추가
git add .

# 커밋
git commit -m "Initial commit: Lab Seminar Archive with AI"

# GitHub에 새 저장소 생성 후 (웹사이트에서)
# 저장소 연결 (YOUR_USERNAME을 본인 계정으로 변경!)
git remote add origin https://github.com/YOUR_USERNAME/lab-seminar-archive.git

# 푸시
git branch -M main
git push -u origin main
```

### 2️⃣ Vercel 배포 (AI 백엔드)

1. **https://vercel.com** 접속
2. **Sign Up** → **Continue with GitHub**
3. **Add New Project** 클릭
4. 방금 만든 저장소 선택 → **Import**
5. **Environment Variables** 추가:
   ```
   Name: ANTHROPIC_API_KEY
   Value: [claudeapi.txt의 내용 복사해서 붙여넣기]
   ```
6. **Deploy** 클릭!
7. 배포 완료 후 URL 복사:
   ```
   예: https://lab-seminar-archive-abc123.vercel.app
   ```

### 3️⃣ config.js 업데이트

Vercel URL을 받았으면:

```bash
cd /raid/lab-seminar
nano config.js
```

다음 줄을 수정:
```javascript
// 이 부분을:
API_BASE_URL: 'https://YOUR_VERCEL_PROJECT.vercel.app/api'

// Vercel URL로 변경:
API_BASE_URL: 'https://lab-seminar-archive-abc123.vercel.app/api'
```

저장 후:
```bash
git add config.js
git commit -m "Update Vercel API URL"
git push
```

### 4️⃣ GitHub Pages 활성화

1. GitHub 저장소 페이지 이동
2. **Settings** → **Pages**
3. **Source**: main branch 선택
4. **Save** 클릭
5. 5분 대기!

## 🔧 자주 묻는 질문

### Q: Vercel 무료인가요?
**A:** 네! 개인 프로젝트로는 무료 티어로 충분합니다.
- 월 100GB bandwidth
- 월 100시간 실행 시간

### Q: GitHub Pages도 무료인가요?
**A:** 네! Public 저장소는 완전 무료입니다.

### Q: API 키가 노출되나요?
**A:** 아니요! API 키는:
- Vercel 환경 변수에만 저장
- `.gitignore`로 GitHub 업로드 차단
- 프론트엔드 코드에 노출되지 않음

### Q: 나중에 코드를 수정하면?
**A:** Git push만 하면 자동 재배포됩니다!
```bash
git add .
git commit -m "수정 내용"
git push
```
